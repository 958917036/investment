## 一、平台页面地图

| 路由 | 页面 | 功能描述 |
|------|------|---------|
| `/` | 首页 Dashboard | 总览、快捷入口、最近分析 |
| `/analyze` | 发起分析 | 单只/批量输入股票，发起L1→L4分析 |
| `/l1` | L1专线 | 快速粗筛，仅L1层过滤 |
| `/result/:id` | 分析详情 | 单次分析完整报告 |
| `/history/:code?` | 历史记录 | 某股票所有历史分析，可对比 |
| `/compare/:code?` | 对比分析 | 两次分析的差异高亮 |
| `/batch` | 批次任务 | 批量分析任务状态管理 |
| `/stocks` | 股票库 | 所有已分析股票一览 |
| `/portfolio` | 持仓 | 已持仓标的跟踪 |
| `/watchlist` | 关注 | 关注标的和目标价 |

## 二、核心用户动线

### 动线A：单只股票完整分析（最常用）

```
首页 / 导航栏
  → 点击「新建分析」
  → 进入 /analyze（单只分析tab，默认）
  → 输入「00700」或「腾讯」
  → 系统自动检测市场（港股）✅
  → 显示股票信息（名称/代码/历史分析次数）
  → 点击「开始分析 · 00700」
  → 自动跳转 /batch?task_id={batch_id}
  → 等待任务完成（进度条 0%→100%）
  → 任务完成 → 点击「查看结果」
  → 进入 /result/{record_id} 详情页
  → 查看L1/L2/L3/L4全部内容
  → 决策：无信号 / 买入信号 / 观察
  → 可选操作：
      · 「加入关注」（设置目标价）
      · 「加入持仓」（填入数量和成本）
      · 「复制链接」（分享给其他人）
      · 「刷新」（用最新数据重新分析）
  → 点击「历史记录」→ /history/00700
  → 可选：勾选两条记录 → 「对比选中项」
  → 进入 /compare/00700 页面
```

### 动线B：批量选股

```
首页 / 导航栏
  → 点击「新建分析」或「批次任务」
  → 进入 /analyze（批量分析tab）
  → 多行输入股票代码：
      00700
      AAPL
      NVDA
      600519
  → 选择市场偏好：自动（默认）
  → 点击「批量分析 · 4只股票」
  → 自动跳转 /batch?task_id={batch_id}
  → 查看批量任务状态：
      · 总计/完成/失败/进行中 实时数字
      · 每只股票的独立状态
      · 失败的可以「重试」
  → 全部完成后，逐个点击「查看结果」
  → 进入各股票的 /result/{id}
```

### 动线C：查看持仓/关注

```
首页
  → 点击「持仓」
  → 进入 /portfolio
  → 查看所有已持仓股票
      · 代码/名称/数量/成本价/现价
      · 盈亏金额和百分比（红绿颜色）
      · 最新分析决策
      · 点击「查看详情」→ /result/{id}
      · 点击「历史记录」→ /history/{code}
      · 可删除持仓

  → 点击「关注」
  → 进入 /watchlist
  → 查看所有关注标的
      · 代码/名称/目标价/关注原因
      · 点击「查看详情」
      · 可编辑/删除关注项
```

### 动线D：L1专线快速筛选

```
L1专线
  → 进入 /l1
  → 输入多个候选股票（每行一个）
  → 选择市场偏好
  → 点击「开始L1筛选」
  → 仅执行L1粗筛层（最快）
  → 查看筛选结果：通过/淘汰
  → 可对通过的标的「发起完整分析」
  → 进入 /batch 流程
```

## 四、快捷按钮清单

| 位置 | 按钮 | 行为 |
|------|------|------|
| 详情页 `/result/:id` | 加入持仓 | 弹窗填数量+成本价，提交 |
| 详情页 `/result/:id` | 加入关注 | 弹窗填目标价+理由，提交 |
| 详情页 `/result/:id` | 复制链接 | 复制当前URL到剪贴板 |
| 详情页 `/result/:id` | 刷新 | 重新拉取最新数据 |
| 详情页 `/result/:id` | 历史记录 | 跳转 /history/{code} |
| 批次页 `/batch` | 重试 | 对失败任务重新执行 |
| 批次页 `/batch` | 查看结果 | 跳转 /result/{id} |
| 批次页 `/batch` | 发起新分析 | 跳转 /analyze |
| 历史记录 `/history` | 对比选中项 | 跳转 /compare/{code} |
| 历史记录 `/history` | 查看详情 | 跳转 /result/{id} |
| 持仓 `/portfolio` | 查看详情 | 跳转 /result/{id} |
| 持仓 `/portfolio` | 删除 | 移除持仓记录 |
| 关注 `/watchlist` | 查看详情 | 跳转 /result/{id} |
| 关注 `/watchlist` | 删除 | 移除关注 |

## 五、数据流向

```
用户输入股票代码
    ↓
analyze API（POST /api/analyze）
    ↓
analyze.py: 创建batch_tasks + analysis_records
    ↓
BackgroundTasks: 触发shennong L1→L4 pipeline
    ↓
各层执行：L1粗筛 → L2技术 → L3量化 → L4决策
    ↓
每层结果写入 analysis_records JSON列
    ↓
更新 stock_profiles 表
    ↓
前端轮询 batch API 或 WebSocket 更新进度
    ↓
用户查看 /result/{id} 详情页
```

## 六、路由参数说明

| 路由 | 参数 | 说明 |
|------|------|------|
| `/result/:id` | record UUID | 分析记录ID |
| `/history/:code?` | stock_code，可选 | 不填则显示全部历史 |
| `/compare/:code?` | stock_code | 股票代码，ids从query string取 |
| `/batch?task_id=` | UUID | 直接定位到某个批次任务 |

## 七、工作模式

| 模式 | 入口 | 说明 |
|------|------|------|
| 单只分析 | `/analyze` (单只tab) | 输入1个股票代码，1个batch任务 |
| 批量分析 | `/analyze` (批量tab) | 输入多个股票，最多50只 |
| L1专线 | `/l1` | 仅执行L1粗筛，最快，适合初筛 |
| 详情查看 | `/result/:id` | 任意record UUID |
| 历史对比 | `/history/:code` + 复选框 | 选两条 → `/compare` |
